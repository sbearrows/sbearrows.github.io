Overview of selected projects with external links to git repository

ddPCR Shiny Web application - show me the code!

Built an R Shiny application for the analysis and quality control of ddpcr data from the Walter lab. This reduces errors by catching incorrect data inputs before they are merged into full dataset, which is shared with collaborators. Additioanlly, I also built Shiny web applications for the analysis of our qPCR data and for the analysis of our collaborators CFU data. My web applications have saved the wet lab members several hours in analysis time.

UFO sightings and US Elections Frequent Itemsets - show me the code!

Used the Apriori Algorithm in the R programming language on a UFO sightings and US Elections dataset (merged on state) to mine for frequent itemsets. My goal was to find any correlations with UFO sightings and political party and or the rise of the internet (pre vs post 1996). This project was mostly for fun and though I did find some interesting patterns in the frequent itemsets in pre vs post 1996, I unfortunately didn't find any correlations in our dataset.

Science Dictionary ADT - show me the code!

Made a Science Dictionary ADT that uses a hash table as the underlying data strucutre, writen in the C++ programming language. This project was inspired by my tenure in academia working on projects in which our collaborators might be in another state and communicating can be difficult without proper documentation (something Computer Scientist are very good at). Often labs will use different names for the same thing or (even better) the same name for different things. I designed this to be easy to use. The underlying hash table automatically resizes if the load factor is â‰¥ 0.6. New variables with definitions are added only if another with the same name or definition does not already exists. The dictionary can be written to a csv file and a Linux-like grep search can be done on the definition portion of the entry. If I were to re-vamp this project I would use an AVL tree instead of a hash table because searching for text using an AVL tree would decrease my run time.
